    Node.js

    Node.js: javascript运行环境(runtime),不同系统直接运行各种编程语言

    从node.js官网下载并安装node，安装过程很简单，只需要修改安装路径，一路“下一步”就可以了（傻瓜式安装）。

    验证Node.js安装成功

    命令行输入node -v, 出现相应的版本号说明安装成功

    Node.js配置

    node.js中自带npm。默认情况下，NPM安装的模块并不会安装到NodeJS的程序目录，而是在C盘某个路径中，如果不修改npm的模块安装目录，那么它默认情况下都会安装到这里，随着测试开发各种不同的项目，安装的模块越来越多，那么这个文件夹的体积会越来越大，直到占满你的C盘。

    1. 修改NPM的缓存目录和全局目录路径，将对应的模块目录改到D盘nodejs的安装目录。
      D盘Nodejs目录下创建node_cache和node_global文件夹，然后在命令行中执行以下两个命令：

    npm config set prefix “D:\Program Files\nodejs\node_global”

    npm config set cache “D:\Program Files\nodejs\node_cache”

    1. 配置npm和nodejs的环境变量，这样nodejs才能正确地调用对应的模块。
    • 在用户环境变量中配置：

    NAME: NODE_PATH
    VALUE: D:\ProgramFiles\nodejs\node_modules\

    • 修改nodejs默认的模块调用路径，在环境变量的Path中寻找是否有一条关于node npm的，修改为：

    D:\Program Files\nodejs\node_global\

    npm

    npm: Node.js下的包管理器。由于国内使用npm会很慢,这里推荐使用淘宝NPM镜像，输入以下命令即可：

    npm install -g cnpm –registry=https://registry.npm.taobao.org

    以后在使用npm的地方就可以用cnpm替代。

    webpack

    它主要的用途是通过 CommonJS 的语法把所有浏览器端需要发布的静态资源做相应的准备，比如资源的合并和打包。

    vue-cli

    用户生成Vue工程模板，俗称脚手架

    运行命令:

    cnpm install vue-cli -g

    构建项目

    1. 找到一个文件位置，打开命令行输入:

    vue init webpack {projectname}

    接下来会提示输入几个值，不想填输回车即可。

    1. 用VS code打开项目。
    2. 在项目路径下打开Terminal中输入以下命令，安装依赖包。

    cnpm install

    1. 运行项目。

    cnpm run dev
